Day 5 Purpose

Students demonstrate that they can identify parts, match symbols, trace paths, compare series and parallel circuits, and use continuity reasoning before moving into Week 5 troubleshooting. The day should feel like a bridge: students are not just learning symbols; they are learning how to use drawings and path logic to diagnose faults.

Safety boundary: Continuity, resistance, and physical tracing checks are done with power disconnected. Student checkoffs use assigned low-voltage trainers or disconnected boards only.

Learning Targets

  • Match common physical components to schematic symbols and labels.
  • Trace source, control point, resistor/protection, load, and return on a schematic and physical board.
  • Explain series and parallel path differences.
  • Use continuity reasoning to identify whether an assigned path is complete or open.
  • Write a technician note that supports Week 5 troubleshooting.
  • Complete quiz corrections and readiness reflection.

Week 4 Quiz

  1. Name five common electronic components and state one purpose for each.
  2. Which components from this week are polarity-sensitive or orientation-sensitive?
  3. What does a resistor do in an LED circuit?
  4. What does a schematic show?
  5. What does R1 usually mean on a schematic?
  6. Why might a schematic look different from the physical build?
  7. Define node in beginner language.
  8. Explain series in your own words.
  9. Explain parallel in your own words.
  10. Why is continuity testing useful for schematic tracing?
  11. Why should power be disconnected before checking continuity?
  12. Describe one difference between physical layout and schematic connection.
  13. What happens when one load opens in a simple series path?
  14. What should you inspect if one branch of a parallel circuit fails but the other branch still works?

Correction Prompts

  • Redraw one schematic path and label source, control, resistor/protection, load, and return.
  • Match five physical parts to their symbols and write each purpose.
  • Explain one series and one parallel example using path language.
  • Write the correct continuity testing safety rule from memory.
  • Rewrite a vague note, such as "circuit has parts," into a technician note with evidence.

Week 5 Readiness Checklist

  • [ ] I can identify common parts and symbols.
  • [ ] I can trace a path from source to return.
  • [ ] I can compare a schematic to a physical circuit.
  • [ ] I can compare series and parallel paths.
  • [ ] I can use continuity reasoning to locate opens.
  • [ ] I can write observations that help with troubleshooting.
  • [ ] I can explain why a visible path problem could create a circuit symptom.

Exit Ticket

Explain how component identification, schematic tracing, and series/parallel reasoning will help you troubleshoot a circuit in Week 5.
