Week 4 Day 4: Series and Parallel Circuits

2-3 hour class block comparing one-path and multi-path circuits.

Week 4Previous DayNext Day

Day 4 Purpose

Students compare series and parallel circuit behavior using simple low-voltage examples. The goal is to understand paths and fault effects, not advanced math. Students should be able to predict what happens when a path opens.

Learning Targets

  • Explain series as one path for current.
  • Explain parallel as more than one path.
  • Predict what happens if one load opens in a simple series or parallel circuit.
  • Use a diagram to identify branches and shared nodes.
  • Document observations from a safe build or demonstration.

Diagram Practice

Series idea:
+ -> LED1 -> LED2 -> -

Parallel idea:
+ -> LED1 -> -
+ -> LED2 -> -

Label the branches and circle the loads.

Build or Demonstration Tasks

  1. Inspect the circuit before applying power.
  2. Trace the series path or parallel branches.
  3. Predict what will happen if one LED path opens.
  4. Run the demonstration or approved build.
  5. Disconnect power before changing any wire.
  6. Record what happened and explain why.

Common Mistakes

  • Thinking parallel means "messy" instead of multiple paths.
  • Assuming every circuit with two loads is parallel.
  • Forgetting that series failures can stop the whole path.
  • Changing wiring while powered during comparison testing.

Knowledge Check and Exit Ticket

1. What does series mean?
2. What does parallel mean?
3. Why might one branch keep working in a parallel circuit?
4. How does continuity testing help find an open in a series path?
Exit ticket: Draw or describe one series path and one parallel branch.
