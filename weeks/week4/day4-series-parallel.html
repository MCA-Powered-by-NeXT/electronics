Student Reading: Learn This First

Series and parallel describe paths. That is the key idea. Do not start by memorizing drawings. Start by asking, “How many paths can current take from source to return?”

A series circuit has one path. If the path opens anywhere, the whole path may stop. It is like a single-lane road with one bridge. If the bridge is out, everyone is stuck. In circuit language, one open point can stop current through the series path.

A parallel circuit has more than one branch path. The branches share source and return points, but current can travel through different branches. If one branch opens, another branch may still work because it has its own path back to the source.

Parallel does not mean “random wires everywhere.” Parallel means branches share two connection points: a source-side node and a return-side node. If the branches do not connect correctly to source and return, they are not useful branches; they are just wire art.

Series and parallel reasoning helps troubleshooting. If two loads in series are both off, an open anywhere in the one path could be the issue. If one branch is off and another branch still works, the fault may be in the failed branch. If all branches are off, inspect shared source or return points first.

Fun fact: holiday lights are a classic way people notice series/parallel behavior. Older strings could fail in annoying ways when one lamp opened. Modern designs often include clever workarounds. The lesson for technicians: path design changes fault symptoms.

Before the activity, complete this sentence: “In series, there is __ path. In parallel, there are __ paths called __.”

Beginner Extra Help: Do Not Count Parts, Count Paths

A circuit with two loads is not automatically parallel. Two loads can be in one series path, or they can be in separate parallel branches. The question is not “How many LEDs do I see?” The question is “How many separate paths connect source to return?”

Use your pencil like a tiny tour guide. Trace from source through the load and back to return. If there is only one route, it is series. If the path splits into branches and comes back together, it is parallel.

Day 4 Purpose

Students compare series and parallel circuit behavior using simple low-voltage examples. The goal is to understand paths and fault effects, not advanced math. Students should be able to predict what happens when a path opens, identify branches, and explain why one load can fail while another still works in a parallel circuit.

Beginner electronics materials often introduce series and parallel circuits with lamps, switches, cells, and resistors. This class uses that same concrete approach with classroom-safe LEDs or trainer lamps and clear source-to-return tracing.

Safety boundary: All builds use assigned low-voltage DC training circuits only. Students disconnect power before opening a path, moving wires, or changing components.

Learning Targets

  • Explain series as one path for current.
  • Explain parallel as more than one branch path sharing source and return points.
  • Predict what happens if one load opens in a simple series or parallel circuit.
  • Use a diagram to identify branches and shared nodes.
  • Explain why series/parallel reasoning helps troubleshooting.
  • Document predictions, observations, and explanations from a safe build or demonstration.

Build or Demonstration Tasks

  1. Inspect the circuit before applying power.
  2. Trace the series path or parallel branches.
  3. Predict what will happen before opening any point.
  4. Run the demonstration or approved build.
  5. Disconnect power before changing or opening any wire.
  6. Record what happened and explain why.
  7. Restore the circuit to the known-good condition.

Common Mistakes

  • Thinking parallel means "messy" instead of multiple paths.
  • Assuming every circuit with two loads is parallel.
  • Forgetting that series failures can stop the whole path.
  • Forgetting that parallel branches still share source and return points.
  • Changing wiring while powered during comparison testing.
  • Drawing separate branches but forgetting to reconnect them to return.

Knowledge Check and Exit Ticket

1. What does series mean?
2. What does parallel mean?
3. Why might one branch keep working in a parallel circuit?
4. How does continuity testing help find an open in a series path?
5. If both parallel branches are off, what shared points should you inspect first?
