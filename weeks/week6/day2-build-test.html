Week 6 Day 2: Build and Initial Test

2-3 hour class block for building the approved final project, inspecting before power, recording measurements, and deciding next troubleshooting steps.

Day 2 Purpose

Students build the approved project using safe work habits and record evidence as they go. The instructor checks that the build matches the approved plan, power is controlled, polarity is correct, protection components are included, and the project can be inspected before first power-up.

The goal is not simply to get an LED or buzzer to work. The goal is to build like a technician: follow a plan, inspect in stages, test safely, record what happened, and identify what needs troubleshooting.

Learning Targets

  • Build according to an approved plan without unapproved scope changes.
  • Use staged building: source section, control section, load section, return section.
  • Inspect before applying power.
  • Record build steps, measurements, observations, and issues.
  • Perform an initial function test safely.
  • Classify the project as working, partially working, or not working yet using evidence.

Stage-Build Method

Students should avoid wiring the entire project at once and then hoping it works. Build in sections and inspect each section.

StageBuild ActionInspection QuestionEvidence To Record
1. SourcePlace or connect approved source leads.Is polarity identified and source approved?Source type and voltage if assigned.
2. ControlAdd switch, button, or control connection.Does the control actually interrupt or complete the intended path?Control point location and expected state.
3. ProtectionAdd resistor or required current-limiting component.Must current pass through the resistor before the LED/load?Resistor value/location.
4. LoadAdd LED, buzzer, lamp, or output.Is polarity correct if the load is polarity-sensitive?Load orientation and label.
5. ReturnComplete path back to source negative/return.Can the path be traced all the way back?Return connection note.
6. InspectCheck wiring, exposed conductor, loose parts, and breadboard rows.Is the build safe enough to test?Pre-power checklist.

Pre-Power Inspection Checklist

  • [ ] Parts match approved plan.
  • [ ] Project uses only approved low-voltage classroom source.
  • [ ] Source polarity identified.
  • [ ] Current-limiting components installed where needed.
  • [ ] Source, control, load, and return can be traced.
  • [ ] LED, buzzer, diode, or other polarity-sensitive parts are oriented correctly.
  • [ ] No loose wires, extra exposed conductor, or unintended paths are visible.
  • [ ] Wiring is clean enough to inspect and explain.
  • [ ] DMM leads and settings checked before any measurement.
  • [ ] Instructor approval before first power-up if required.

Common Build Mistakes

  • Building before the plan is approved.
  • Forgetting the return path.
  • Reversing LED, buzzer, diode, or source polarity.
  • Leaving out a current-limiting resistor or placing it outside the actual path.
  • Putting jumpers in adjacent breadboard rows instead of the same row.
  • Using a switch terminal pair that does not control the intended path.
  • Changing several things without updating the documentation.
  • Calling a project "broken" before recording the actual symptom.

Exit Ticket

Describe the current state of your project: working, partially working, not working yet, or unsafe/unclear. Include one piece of evidence and the first step you will take next class.
