Week 6 Day 1: Final Project Planning

2-3 hour class block for choosing, scoping, sketching, approving, and documenting a safe low-voltage final project.

Day 1 Purpose

Students prepare for the final project by selecting or receiving an approved low-voltage project, identifying required parts, drawing the expected circuit path, planning safe build steps, and deciding how the finished project will be tested. The planning day prevents random building and makes the final project feel like real technician work: plan, build, test, troubleshoot, document, and explain.

Beginner electronics references often start with simple useful circuits: a battery, conductors, a switch or sensor, a resistor, and an output such as a lamp, LED, or buzzer. Week 6 final projects should stay at that level of complexity. Students may add polish, but they should not add unapproved power sources, complex modules, or unknown equipment.

Safety boundary: Final projects must be approved low-voltage DC projects. Students do not change project scope, source type, or wiring plan without instructor approval.

Learning Targets

  • Select or receive an approved low-voltage final project.
  • Describe the project purpose in one clear sentence.
  • Create a parts list with source, control, load, resistor/protection, wires, and mounting needs.
  • Identify source, path, control point, load, return, and expected operation.
  • Write pre-build safety checks and first-test steps.
  • Define what evidence will prove the project works.
  • Submit a plan for instructor approval before building.

Planning Model: From Idea To Buildable Project

Project idea: Pushbutton buzzer alarm

Function: Buzzer sounds only while the button is pressed.
Expected path: + source -> pushbutton -> buzzer -> return -> - source
Control point: pushbutton
Load/output: buzzer
Evidence: source voltage, correct polarity, buzzer sounds when pressed, silent when released
Safety: approved source only; power disconnected before rewiring

Peer Review Checklist

  • [ ] I can identify the source, control point, load, and return in my partner's plan.
  • [ ] The plan uses an approved low-voltage source.
  • [ ] The plan has a resistor or protection where needed.
  • [ ] The plan says what should happen when it works.
  • [ ] The plan says how to test the project safely.
  • [ ] The plan has a documentation requirement.
  • [ ] I found one possible build mistake for my partner to watch for.
