Student Reading: Learn This First

Start the demonstration by naming the project purpose. The audience should know what the project is supposed to do before seeing it operate.

Point to the source, control, protection or current limiting, load, and return. Do not only point to the output; explain the full path.

Show the test evidence. This may be a function test, measurement note, continuity note, or troubleshooting record.

Explain one problem solved during the build, even if it was small. A project with documented troubleshooting shows technician thinking.

Answer follow-up questions using evidence. If asked why the LED is protected, point to the resistor in the path. If asked how the repair was verified, describe the retest.

End by stating one maintenance or improvement note for future use.

Lesson Work

  1. Demonstrate final project operation.
  2. Explain source, control, protection, load, and return.
  3. Show documentation evidence.
  4. Answer instructor follow-up questions.
  5. Complete final demonstration rubric.
