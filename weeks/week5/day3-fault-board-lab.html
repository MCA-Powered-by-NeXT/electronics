Student Reading: Learn This First

A fault board is a safe training system with a planned fault. The instructor knows the fault; the student practices finding evidence.

Begin with expected operation. What should the board do if it works? Which source, control, load, and return are involved?

Observe the symptom. Describe exactly what happens: no output, dim output, intermittent output, switch not controlling output, or unexpected behavior.

Make the circuit safe before touching wiring or using continuity. Safety is part of troubleshooting, not a separate step.

Choose a likely section to inspect or test. If all output is off, start with source or shared return. If one branch is off, compare that branch to a working branch.

End with repair ticket documentation. The ticket should be specific enough that another technician could understand the symptom, evidence, correction, and verification.

Lesson Work

  1. Record expected operation before touching the board.
  2. Observe and write the symptom.
  3. Make the board safe, then inspect and test one section.
  4. Correct one issue, verify operation, and write the repair ticket.
