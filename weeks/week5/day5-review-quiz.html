Week 5 Day 5: Review, Quiz, and Documentation Cleanup

2-3 hour class block for troubleshooting review, quiz, correction, and final project readiness.

Week 5Previous DayWeek 6

Day 5 Purpose

Students review troubleshooting habits and complete documentation before beginning the final project week. The goal is to leave Week 5 with a reliable troubleshooting process that students can use independently in Week 6.

Review Targets

  • Control and workmanship vocabulary.
  • Troubleshooting flow.
  • Fault categories.
  • Repair ticket evidence.
  • Safe low-voltage checkoff habits.
  • Verification after correction.

Week 5 Quiz

  1. What is the first thing a technician should record when troubleshooting?
  2. Why should a technician change only one thing at a time?
  3. Name four fault categories.
  4. Why does workmanship affect troubleshooting?
  5. What belongs in a repair ticket?
  6. What safety action comes before rewiring?
  7. How does source-path-load-return reasoning help fault isolation?
  8. How do you verify that a repair worked?
  9. What is the difference between a symptom and a cause?
  10. Write a complete repair note for a missing return wire fault.

Documentation Cleanup

  • [ ] Symptom clearly written.
  • [ ] Safety steps recorded.
  • [ ] Evidence included.
  • [ ] Correction stated.
  • [ ] Verification result included.
  • [ ] Note is clear enough for another technician to understand.

Final Project Readiness Checklist

  • [ ] I can plan a safe low-voltage build.
  • [ ] I can troubleshoot without guessing.
  • [ ] I can document build steps and corrections.
  • [ ] I can explain my work to an instructor.
  • [ ] I can verify a result instead of assuming the repair worked.

Exit Ticket

Write the troubleshooting sequence and explain which step prevents guessing.
